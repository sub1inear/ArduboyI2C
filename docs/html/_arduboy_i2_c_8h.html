Macro Definition Documentation

◆ I2C_FREQUENCY

#define I2C_FREQUENCY   100000

The initial I2C frequency in Hz.

Defaults to 100000 Hz, with a maximum of 300000 Hz. The Arduboy FX-C only has software pullups, so the I2C frequency must be < 300000 Hz to ensure reliability.

◆ I2C_BUFFER_CAPACITY

#define I2C_BUFFER_CAPACITY   32

The capacity of the buffer used for writes/target (slave) operations.

Defaults to 32. If more than 32 bytes are needed for writes/target (slave) operations, increase. If more RAM is needed, decrease. Maximum is 255.

◆ I2C_TARGET_ADDRESS

#define I2C_TARGET_ADDRESS   0x08

The target address set by the handshake function.

This is the address that the controller (master) should use to communicate with the target (slave) when using the handshake function. This is optional; the address can be changed to whatever you wish but is kept for convenience.

◆ I2C_NULL_ADDRESS

#define I2C_NULL_ADDRESS   0x09

The null address used to disable a target from responding.

This is useful in lockstep multiplayer to soft-disable a target (slave) from responding to the controller (master) without having to call I2C::end(). This is only a convention and is optional; the address can be changed to whatever you wish but is kept for convenience.
